Как ведет себя стрелочная функция, если её вызвать как метод объекта?
let obj = {
	value:42,
	getValue: () => this.value
}
console.log(obj.getValue());

правильный ответ:
++ 		: Вернёт undefined

ошибки:
-- Вернёт this
-- Ошибка ReferenceError
-- Вернет объект obj
-- Вернет 42

* В JavaScript стрелочные функции не имеют собственного контекста this. Они «захватывают» значение this из лексического окружения (родительской области видимости) в момент своего создания. В этом примере объект создаётся в глобальной области видимости (или в модуле). Поэтому this внутри стрелочной функции будет ссылаться на глобальный объект (например, window в браузере), а не на сам объект obj. Так как у глобального объекта нет свойства value, выражение вернёт undefined.

Какой результат выполнения будет у следующего кода?
const user = {info: {id: 123, status: 'active'}};
const {info: {status}} = user;
console.log(status);

правильный ответ:
++ 		: 'active'

ошибки:
-- undefined
-- {status: 'active'}
-- SyntaxError
-- 123

В JavaScript используется механизм деструктуризации объектов. В выражении const {info: {status}} = user; происходит следующее:Код заглядывает внутрь объекта user и находит свойство info.Затем внутри этого вложенного объекта info он находит свойство status.Значение этого свойства ('active') извлекается и записывается в одноименную константу status.В результате в консоль будет выведена строка active.

Каково основное правило, описывающее доступ к параметрам внутри функции в JavaScript?

правильный ответ:
++ 		: Они локальны внутри функции

ошибки:
-- Они передаются в объекты при помощи this
-- Они сохраняются в глобальном объекте
-- Они скрыты от замыканий
-- Они доступны во всём приложении

В JavaScript лексическое окружение (то есть переменные, доступные внутри функции) «привязывается» к самой функции в момент её объявления.
Замыкание — это функция, которая «помнит» свои внешние переменные и имеет доступ к ним даже после того, как внешняя функция, в которой они были созданы, завершила свою работу. В JavaScript каждая функция создает замыкание при своем создании.

Для чего используется event.preventDefault() в обработчиках событий?

правильный ответ:
++ 		: Предотвращает стандартное действие браузера для события

ошибки:
-- Останавливает распространение событий вверх по DOM
-- Останавливает выполнение кода дальше по цепочке
-- Разрешает всплытие события
-- Удаляет элемент из DOM

* Всплытие события (Event Bubbling) — это механизм в JavaScript, при котором событие, произошедшее на вложенном элементе (например, клик по кнопке), автоматически передается вверх по иерархии DOM-дерева до самого верхнего элемента (window или document)

Остановка всплытия: Если вы хотите, чтобы событие сработало только на дочернем элементе и не передавалось родителям, в обработчике нужно вызвать метод event.stopPropagation()

Что будет результатом кода и почему?
function Shape() {}
Shape.prototype.area = function() { return 42; };
const s = new Shape();
Shape.prototype = { area() { return 99; } };
console.log( s.area() );

правильный ответ:
++ 		: 42

ошибки:
-- null
-- 99
-- undefined
-- ошибка выполнения

В JavaScript объекты наследуют методы из прототипов, а связь между объектом и его прототипом устанавливается в момент создания объекта

У функции Shape появляется свойство prototype, внутри которого хранится метод area (возвращающий 42).
Создаем объект s. В этот момент движок JavaScript связывает созданный объект s со старым прототипом Shape.prototype. Внутренняя ссылка __proto__ объекта s жестко указывает на объект { area: function() { return 42; } }.
Мы полностью перезаписали объект, на который ссылается свойство Shape.prototype. Теперь конструктор Shape создает новые объекты с методом, возвращающим 99.
Когда мы вызываем метод у объекта s, JavaScript ищет метод area сначала в самом объекте s, а затем в его прототипе. Поскольку ссылка объекта s на его прототип осталась прежней (указывает на старый прототип с 42), код успешно вернет 42

Как можно изменить прототип объекта в JavaScript?

правильный ответ:
++ 		: С помощью Object.setPrototypeOf(obj, prototype)

ошибки:
-- Прототип нельзя изменить после создания
-- Прототип заменяется автоматически при вызове конструктора
-- Только при помощи классов
-- Нужно вызвать prototype.override()

__proto__ — это встроенное свойство объекта в JavaScript, которое указывает на его прототип (другой объект, от которого данный объект наследует свойства и методы). По сути, это скрытая ссылка, лежащая в основе механизма прототипного наследования.
Когда вы пытаетесь прочитать свойство объекта, но его там нет, JavaScript автоматически ищет это свойство в его __proto__, затем в __proto__ этого прототипа, и так далее (эта цепочка называется цепочкой прототипов)
Важное отличие: __proto__ и prototype
__proto__ — это свойство конкретного объекта, которое указывает, от кого он унаследовал свойства.
prototype — это свойство функции-конструктора (или класса), которое задает __proto__ для всех объектов, созданных этой функцией с помощью оператора new
Использование __proto__ в коде считается устаревшим.

Что произойдет при выполнении кода?
function createCounter() {
	let count = 0;
	return {
		increment() {
			count++;
			return count;
		}
	};
}
const counter = createCounter();
console.log( counter.increment() );

правильный ответ:
++ 		: Метод increment использует замыкание для доступа к count и увеличивает его

ошибки:
-- Функция createCounter ничего не возвращает
-- Создается глобальная переменная count и метод increment в counter, который увеличивает count
-- Каждый вызов increment возвращает 1, так как заново создается count и увеличивается на 1
-- count становится недоступным после первой итерации
