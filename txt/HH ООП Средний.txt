Какое из утверждений верно?

правильный ответ:
++ 		: Класс - это шаблон, определяющий состояние и поведение объектов, объект - экземпляр класса

ошибки:
-- Объект - это функция, класс - значение, возвращаемое этой функцией
-- Объект - это шаблон, определяющий состояние и поведение классов, класс - экземпляр объекта
-- Класс - это имя переменной, объект - значение, которое присваивается этой переменной
-- Класс - это функция, объект - значение, возвращаемое этой функцией

Какой из вариантов наиболее точно описывает проблему работы с антипаттерном God Object в ООП?


++ 		: Объект управляет всеми процессами системы, нарушая принцип разделения ответственности

ошибки:
-- Перегрузка системы из-за избыточного использования памяти в ходе обработки всех процессов
-- Создание слишком большого количества объектов для выполнения ззадач и усложнение работы с методами
-- Использование глобальных переменных для управления объектами приводит к проблемам уровней доступа
-- Полиморфизм объекта нарушает целостность программы и работы с атрибутами

Какова основная цель абстракции и инкапсуляции?

правильный ответ:
++ 		: Абстракция упрощает объектную модель, инкапсуляция защищает данные от изменения

ошибки:
-- Абстракция ограничивает доступ к методам, инкапсуляция скрывает атрибуты объектов
-- Абстракция скрывает всю функциональность, инкапсуляция улучшает производительность
-- Абстракция предоставляет общий доступ к методам, инкапсуляция объединяет объекты в группы
-- Абстракция работает только с интерфейсами, инкапсуляция - с классами и объектами

Когда наиболее целесообразно применять статический полиморфизм?

правильный ответ:
++		: Когда все типы данных и их комбинации, передаваемые в функцию или метод, известны на этапе компиляции

ошибки:
-- Когда функции требуют динамического выбора между разными реализациями
-- Когда необходимо использовать одиин и тот же метод для разных объектов
-- Когда код должен адаптироваться к различным платформам и средам разработки
-- Когда методы должны изменяться в зависимости от времени выполнения кода
