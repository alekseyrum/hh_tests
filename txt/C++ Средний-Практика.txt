Фрилансеры со всего света
Средний Валидация данных Коллекции Обработка даты и времени

Отдел управления персоналом ИТ-компании формирует отчет для департамента по налогам по количеству внештатных сотрудников в разных странах за прошлый год в формате:
Фамилия;ГодРождения;НазваниеСтраны;ProjectID.
Требуется проверить корректность данных:
*Фамилия содержит только символы английского алфавита без пробелов и других символов, длина поля - от 1 до 40 символов включительно.
*ГодРождения - целое число от 1960 до 2007 включительно.
*НазваниеСтраны содержит только символы английского алфавита без пробелов и других символов, длина поля - от 2 до 40 символов включительно, название страны не равно none.
*ProjectID имеет формат CCNNN, где С - это символ английского алфавита в верхнем регистре, N - целое число от 0 до 7 включительно, например AF470.

На основе корректных данных нужно вывести список стран и количество внештатных сотрудников из каждой страны.
Список стран нужно упорядочить по убыванию количества внештатных сотрудников, при равенстве значений количества фрилансеров упорядочить НазваниеСтраны по алфавиту.
Требуется разработать класс ProcessingFreelancers, который содержит метод ValidateFreelancer для определения корректности данных о внештатных сотрудниках, и GenerateFreelancersReport, который возвращает список стран с количеством внештатных сотрудников для каждой страны в формате
Нзвание страны:КоличествоСотрудников.

ФОРМАТ ВВОДА
На вход подаются строки, содержащие информацию в формате:
Фамилия;ГодРождения;НазваниеСтраны;ProjectID.
Строка содержит ровно четыре поля, разделенных ;.
До или после разделителя ; допускаются пробелы.
Количество строк - от 1 до 1000. Гарантируется, что есть хотя бы одна строка.

Вывести список стран с количеством внештатных сотрудников для каждой страны в формате Название страны:КоличествоФрилансеров.
Если корректных данных о внештатных сотрудниках нет, вывести:

std::vector<std::string> generate_report(const std::vector<std::string> & lines) 
{
    std::vector<std::string> vecResult;
    std::map<string, int> mapWorkers;

    for (const std::string &word : lines) {
        std::string_view sv(word);
        int delim1 = sv.find(';');
        int delim2 = sv.find(';', delim1+1);
        int delim3 = sv.find(';', delim2+1);
        if (delim3 == std::string_view::npos) {
            continue;
        }
        std::string_view svName = sv.substr(0,delim1);
        std::string_view svYear = sv.substr(delim1+1, delim2-delim1-1);
        std::string_view svCountry = sv.substr(delim2+1, delim3-delim2-1);
        std::string_view svProject = sv.substr(delim3+1, sv.size()-delim3);
        // cout << "name:" << svName << " year:" << svYear << " contry:" << svCountry << " proj:" << svProject << endl;
        if ( svName.size() > 40 || svCountry.size() > 40 || svCountry.size() < 2) { 
            cout << "wrong field length:" << svName << " or " << svCountry << endl;
            continue; 
        }
        if ( ! std::all_of(svName.begin(), svName.end(), [](unsigned char c) { return std::isalpha(c); } ) )  {
            cout << "wrong name:" << svName << endl;
            continue; 
        }
        int iYearVal = 0;
        std::from_chars_result fcr = std::from_chars(svYear.data(), svYear.data() + svYear.size() , iYearVal );
        if ( iYearVal < 1960 || iYearVal > 2007) {
            cout << "wrong year:" << iYearVal << endl;
            continue;
        }

        if ( ! std::all_of(svCountry.begin(), svCountry.end(), [](unsigned char c) { return std::isalpha(c); } ) )  {
            cout << "wrong country:" << svCountry << endl;
            continue; 
        }
        // - - - [field ProjectID] - - -
        if ( svProject.size() != 5 || ! std::all_of(svProject.begin(), svProject.begin()+2, [](unsigned char c) { return (c >= 'A' && c <= 'Z');} ) ) {
            cout << "wrong project format:" << svProject << endl;
            continue;
        }    
        if ( ! std::all_of(svProject.begin()+3, svProject.begin()+5, [](unsigned char c) { return std::isdigit(c); } ) ) {
            cout << "wrong project format:" << svProject << endl;
            continue;
        }   
        // here all ok:
        std::map<string,int>::iterator itWorkers = mapWorkers.find( std::string(svCountry) );
        if ( itWorkers == mapWorkers.end() ) {
            mapWorkers.insert( { std::string(svCountry), 1 } );
        } else {
            itWorkers->second++;
        }
    }

    if ( mapWorkers.size() == 0 ) {
        return {"none:0"};
    }

    std::vector<std::pair<std::string, int>> vec4Sort(mapWorkers.begin(), mapWorkers.end());
    std::sort(vec4Sort.begin(), vec4Sort.end(), [](const auto& a, const auto& b) {
        if (a.second != b.second) return a.second > b.second;
        return a.first < b.first;
    });
    for (auto cc : vec4Sort) {
        vecResult.push_back( cc.first + ":" + std::to_string(cc.second) );
    }
    return vecResult;
}

std::vector<std::string> report(const std::vector<std::string> & lines) 
{
    std::vector<std::string> vecResult;

    for (auto cc : lines) {
        std::string_view sv(cc);

        int delim1 = sv.find("->");
        int delim2 = sv.find("->", delim1+2);
        int delim3 = sv.find("->", delim2+2);
        if (delim3 == std::string_view::npos) {
            cout << "fields too few" << endl;
            return vecResult;
        }
        std::string svGameid(sv.substr(0,delim1) );
        std::string svName(sv.substr(delim1+2, delim2-delim1-2) ) ; 
        std::string svRate(sv.substr(delim2+2, delim3-delim2-2) );
        std::string svLoad(sv.substr(delim3+2, sv.size()-delim3) );
       
        int iGameID = 0 ; 
        if ( svGameid.empty() ) { 
            svGameid = "unknown" ;
        } else {
            iGameID = std::stoi(svGameid);
        }
        
        int iRate = std::stoi(svRate);
        int iDownLoad = std::stoi(svLoad);
        std::string sMsg = "low";

        if (iGameID < 1000 || iGameID > 9999 || svName.length() < 5 || svName.length() > 40 
            || ! std::all_of(svName.begin(), svName.end(), [](unsigned char c) { return std::isalpha(c); })
            || iRate < 0 || iRate > 100
            || iDownLoad < 0 || iDownLoad > 10000000
            ) { 
            if ( svName.empty() ) { svName = "unknown"; }
            vecResult.push_back(svGameid + ":" + svName + ":incorrect data");
        } else {
            if (iRate >= 90 || iDownLoad > 100000) { sMsg = "top"; }
            else if ( (iRate > 60 && iRate <= 89) || (iDownLoad > 50000 && iDownLoad < 99999 ) ) 
            { sMsg = "middle"; }
            vecResult.push_back(svGameid + ":" + svName + ":" + sMsg);
        }         
    }  
    return vecResult; 
}

Вы разрабатываете модуль анализа активности системных служб в корпоративной информационной системе на основе исследования логов запросов, поступающих на сервер.
Запись лога имеет следующий вид:
<service="serviceID" data="dataID" action="readwrite">.
Нужно определить корректные записи логов сервера.
Корректные записи обрабатываются следующим образом:
*Для операций read и write для каждого сервиса производится подсчет сделанных запросов.
*Операция write - для каждого сервиса производится подсчет количества сделанных запросов, а также детектируется наличие подозрительной активности: количество операций write >= 75% от общего количества операций readwrite.
Для каждого сервиса нужно вывести количество операций read и write либо сообщение о подозрительной активности.
Если корректные записи логов отсутствуют, вывести none.
Требуется разработать класс ServerLogAnalyzer с методом ProcessingServerLogs, который выводит информацию об активности сервисов в требуемом формате.

#include <iostream>
#include <vector>
#include <map>
#include <regex>

using namespace std;

struct OpStatus {
    int read;
    int write;
};

std::vector<std::string> report(const std::vector<std::string> & lines) 
{
    std::vector<std::string> vecResult;
    std::map<string, OpStatus> mStates;
    std::map<string, OpStatus>::iterator itElem;

    std::regex pattern("^<service=\"(\\d{5})\" data=\"(\\w{9})\" action=\"(\\w{4,5})\">"); 
    std::smatch matches;

    std::vector<std::string>::const_iterator itLine = lines.begin();
    while ( itLine != lines.end() )
    {
        if (std::regex_search( *itLine , matches, pattern)) {
            itElem = mStates.find( matches.str(1) );
            if ( itElem == mStates.end() ) {
                OpStatus os;
                os.read = 0;
                os.write = 0;
                auto pIns = mStates.insert( { matches.str(1) , os } );
                itElem = pIns.first;
            } 
            if ( matches.str(3) == "read") {
                itElem->second.read++;
            } else if (matches.str(3) == "write") {
                itElem->second.write++;
            }
        }
        itLine++;
    }
    // write-out to result:
    for (const auto & dd : mStates) {
        int iWriteLimit = (dd.second.write + dd.second.read) *0.75;
        if ( dd.second.write >= iWriteLimit) {
            vecResult.push_back( "Alert! " + dd.first + " has suspicious activity" );
        } else {
            vecResult.push_back( "{service=\"" + dd.first + "\",\"read\":" + std::to_string(dd.second.read) + ",\"write\":" + std::to_string(dd.second.write) + "}" );
        }
    }
    if ( vecResult.size() == 0 ) {
        vecResult.push_back( "none" );
    }
    return vecResult;

Мониторинг акций
Средний Делегаты и события Принципы SOLID Коллекции

Вы разрабатываете систему мониторинга изменений цен акций отдела ценообразования в большой компании-ритейлере.
Каждое событие изменения цены акции имеет формат:
*Symbol::Price::ChangeDirection::Timestamp.
*Symbol - тикер акции (например AAPL).
*Price - текущая цена (целое число).
*ChangeDirection - направление изменения: UP (рост) или DOWN (падение).
*Timestamp - время изменения в формате ЧЧ:ММ:СС.
Правила обработки:
Если тикер акции встретился первый раз, он заносится в список тикеров акций.
Если тикер акции уже есть, то происходит обновление цены по следующим правилам:
*Если цена выросла (UP) и временная метка позже сохраненной, то вывести:
{"Symbol":"AAPL","Price":150,"Trend":"UP"}.
*Если цена упала (DOWN) и временная метка позже сохраненной, то вывести:
Symbol DOWN priceChange,
где priceChange - разница между предыдущей и текущей ценой акции.
*Если изменений цены акции не было, вывести none.
Нужно разработать класс SharePriceSystem, реализующий обработку изменения цены акции через подписку на события.

ФОРМАТ ВВОДА
На стандартный ввод подаются строки, содержащие информацию о цене акций:
Symbol::Price::ChangeDirection::Timestamp.
Поля Symbol::Price::ChangeDirection::Timestamp всегда корректные и не пустые и не содержат пробелов.
Значение поля Symbol - четыре символа английского алфавита в верхнем регистре.
Значение поля Price - целое число от 1 до 9999 включительно.
Значение поля ChangeDirection - одно из двух значений (UP или DOWN).
Значение поля Timestamp - время в формате ЧЧ:ММ:СС - часы от 00 до 23 включительно, ММ - минуты от 00 до 59 включительно, СС - секунды от 00 до 59 включительно).
Количество строк от 1 до 1000. Гарантируется, что есть хотя бы одна строка.

ФОРМАТ ВЫВОДА
Сообщения о предложениях цены объектов недвижимости.
Если цена выросла (UP), вывети:
{"Symbol":"AAPL","Price":150,"Trend":"UP"}.
Если цена упала (DOWN), вывести:
Symbol DOWN price.
Если изменений цены акции не было, вывести none.
