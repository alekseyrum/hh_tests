В директории /var/log вы хотите найти все текстовые файлы с расширением .log, в содержимом которых есть слово timeout, и получить список их имен. Какую команду используете?

правильный ответ:
++ 		: find /var/log -type f -name "*.log" -exec grep -l "timeout" {} \;

ошибки:
-- find /var/log -name "*.log" | grep -r "timeout"
-- grep -rl "timeout" /var/log -exec find -name "*.log"
-- find /var/log -type f -grep "timeout" > *.log
-- grep -l "timeout" $(find /var/log -type d -name "*.log")

По совету ГуглИИ: find /var/log -type f -name "*.log" -exec grep -l timeout {} \;
Но так тоже получается: find /var/log -name "*.log" | grep -r "timeout"
Или так: grep -rl "timeout" /var/log/*.log ,но про это ГуглИИ написал, что будут пропущены подкаталоги, содержащие *.log

{} — это плейсхолдер (заполнитель). Вместо него find подставляет пути к найденным файлам.
+ — это индикатор группировки. Он приказывает утилите find собрать все найденные файлы в один большой список и передать их команде grep за один раз.
[Сравнение с {} ;]
Вариант {} + (Группировка) выполняется: grep -l "timeout" файл1.log файл2.log файл3.log ...
Вариант {} \; (Поочередно) выполняется: find запускает отдельный процесс grep для каждого файла:
	grep -l "timeout" файл1.log
	grep -l "timeout" файл2.log
	...
Синтаксис:
find [где_искать] [критерии_поиска] -exec [команда] {} [способ_завершения]
способ_завершения - может быть только одним из двух: + или \;

Проанализируйте следующий скрипт и выберите корректное объяснение его поведения:
#!/bin/bash
count=1
while [ $count -le 5 ]; do
	echo $count
done 

правильный ответ:
++ 		: Скрипт зациклится и никогда не завершится

ошибки:
-- count увеличивается на каждом шаге
-- Переменная count не определена
-- Скрипт ничего не выведет
-- Скрипт корректно выведет числа от 1 до 5
