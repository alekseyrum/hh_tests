Укажите, какая строка должна быть на месте пропуска, чтобы исключение было корректно обработано и в консоль было бы выведено S:
class GeneralException : public std::exception {
public:
    virtual void print() const { std::cout << "G"; }    
};
class SpecialException : public GeneralException {
public:
    void print() const override { std::cout << "S"; }    
};
void f() { throw SpecialException(); }
int main() {
    try {
        f();
    }
    _____
    catch (const SpecialException &e) {
        e.print();
    }
}

правильный ответ:
++      : catch (const GeneralException & e) { e.print(); } 

ошибки:
-- catch (const GeneralException &e) { e.GeneralException::print(); }
-- catch (const std::exception &e) { dynamic_cast<const GeneralException&> (e).print(); }
-- catch (const GeneralException e) { e.print(); }
-- catch (GeneralException e) { e.print(); }

* Перехват по ссылке: Благодаря ключевому слову const GeneralException & программа вызывает виртуальный метод print() у фактического объекта (SpecialException), за счет чего в консоль успешно выводится S вместо G. Если бы вы ловили исключение по значению, сработала бы срезка (slicing) и вызвался бы метод базового класса.
* Срезка (slicing) в C++ — это потеря производной части объекта при копировании его в базовый класс. Когда вы ловите исключение по значению, а не по ссылке, программа создает совершенно новый объект базового типа, отсекая все поля и методы, которые были добавлены в производном классе.

Код ниже не может быть скомпилирован из-за проблем с определением шаблона. Какая проблема определения шаблонных функций мешает работоспособности этого решения?
template<class T1, class T2>
void f() 
{ 
    std::cout << "f() <class T1, class T2>\n"; 
}
template<>
void f<int, int>() { 
    std::cout << "f() <int, int> \n"; 
}
template<class T1>
void f<T1, double> { 
    std::cout << "f() <T1, double>\n"; 
}

правильный ответ:
++      : Нельзя создавать частичную сециализацию функции

ошибки:
?? Неправильное декларирование частичной специализзации функции
-- Нельзя создавать полную специализацию функции
-- Нельзя создавать ни полную, ни частичную специализацию функции
-- Неправильное декларирование полной специализации функции

C++ запрещена частичная специализация шаблонных функций
* Частичная специализация функции (void f<T1, double>):Последняя строка является попыткой частично специализировать шаблонную функцию (один параметр T1 остается обобщенным, а второй double жестко задан). Стандарт C++ не поддерживает частичную специализацию для функций любого типа (как обычных, так и методов классов). Компилятор выдаст ошибку.

Как объявить функцию-корутину с С++20?

правильный ответ:
++      : task<int> foo() co_return 

ошибки:
-- task<int> foo() co_await
-- coroutine std::function<int()> foo()
-- std::coroutine int foo()
-- function std::coroutine<int()> foo()

*Функция в C++20 становится корутиной, если её тело содержит хотя бы одно из ключевых слов: co_await, co_yield или co_return. Они являются "бесстековыми" (stateful) — компилятор преобразует их в конечный автомат, а состояние сохраняет в куче.

Зачем в приведенном коде используется std::atomic ?
#include <iostream>
#include <thread>
#include <atomic>
std::atomic<int> counter(0);
void increment() {
    for(int i=0; i<10000; ++i) { counter.fetch_add(1); }
}
int main() {
    std::thread t1(increment), t2(increment);
    t1.join(); t2.join();
    std::cout << counter << std::endl;
    return 0;
}

правильный ответ:
++      : Использование atomic гарантирует корректное значение counter без состояни гонки

ошибки:
-- atomic используется по умолчанию
-- atomic замедляет программу для предотвращения возникновения тупика
-- atomic требует дополнительной блокировки мьютекса для предотвращения состояния гонки
-- atomic не требует дополнительной блокировки мьютекса для предотвращения состояния голодания

Выберите из предложенных вариантов тот, который правильно описывает поведение данного кода:
#include <mutex>
#include <thread>
std::mutex mtx1, mtx2;
void threadFunction() {
    mtx1.lock();
    mtx2.lock();
    std::lock_guard<std::mutex> lock1(mtx1, std:adopt_lock);
    std::lock_guard<std::mutex> lock2(mtx2, std:adopt_lock);
    // Критическая секция
}
int main() {
    std::thread t1(threadFunction);
    std::thread t2(threadFunction);
    t1.join();
    t2.join();
    return 0;
}

правильный ответ:
++      : Возможен deadlock, потому что мьютексы блокируются по отдельности

ошибки:
-- Корректная работа без deadlock, так как мьютексы захватываются атомарно с помощью std::lock
-- Программа аварийно завершится, так как std::lock не поддерживает несколько мьютексов
-- Ошибка компиляции из-за неправильного использования std::lock_guard
-- Возможно состояние голодания из-за использования нескольких мьютексов

* Для предотвращения взаимных блокировок в C++ следует использовать функцию std::lock(), которая захватывает несколько мьютексов одновременно (атомарно).Чтобы исправить код и обеспечить корректную работу без deadlock, следует переписать threadFunction следующим образом:
void threadFunction() {
    std::lock(mtx1, mtx2); // Атомарный захват обоих мьютексов
    std::lock_guard<std::mutex> lock1(mtx1, std::adopt_lock);
    std::lock_guard<std::mutex> lock2(mtx2, std::adopt_lock);    
    // Работа с защищенными данными
}

Какой интринсик вычисляет горизонтальную сумму 4-х float-элементов в __m128?
__m128 v = _mm_set_ps(1.0f, 2.0f, 3.0f, 4.0f);
float sum = _____;

правильный ответ:
++      : _mm_cvtss_f32(_mm_hadd_ps(_mm_hadd_ps(v,v), _mm_hadd_ps(v,v))) 

ошибки:
-- _mm_mvtss_f32(_mm_add_ps(v, _mm_shuffle_ps(v,v, _MM_SHUFFLE(0,1,2,3))))
-- _mm_cvtss_f32(_mm_hadd_ps(v, v))
-- _mm_cvtss_f32(_mm_dp_ps(v, _mm_set1_ps(1.0f), 0xF1))
-- _mm_cvtss_f32(_mm_add_ps(v, _mm_shuffle_ps(v, v, _MM_SHUFFLE(0,1,2,3))))

*Интринсики (от англ. intrinsics или intrinsic functions) — это встроенные функции языка программирования, которые выглядят и вызываются как обычные функции, но на самом деле обрабатываются компилятором особым образом.
_mm_add_ps - сложение
_mm_cvtss_f32 - извлекает первый элемент из вектора
_mm_hadd_ps (v1, v2) - горизонтальное сложение (HAddd) двух векторов
_mm_dp_ps (v1, v2, mask) - скалярное произведение (DotProduct) двух векторов
_mm_shuffle_ps(v1, v2, mask) - извлечение из векторов элементов и сборка нового вектора
